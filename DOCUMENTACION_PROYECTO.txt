1. OBJETIVO DEL PROYECTO
================================================================================

Este sistema es una plataforma web completa para la gestión de proyectos, 
actividades y tareas dentro de una organización. Permite a los equipos 
colaborar de manera eficiente, gestionar permisos granulares, realizar 
seguimiento de avances, almacenar evidencias de trabajo y mantener un 
historial completo de cambios (trazabilidad).

El sistema está diseñado para:
- Centralizar la gestión de múltiples proyectos simultáneos
- Facilitar la colaboración entre equipos y departamentos
- Implementar controles de acceso basados en roles y permisos
- Registrar y almacenar evidencias del trabajo realizado
- Proporcionar métricas e indicadores de rendimiento
- Mantener un registro completo de todas las acciones realizadas

4. FUNCIONALIDADES PRINCIPALES
================================================================================

4.1 GESTIÓN DE PROYECTOS
-------------------------
Funcionalidades:
- Crear, editar, eliminar y listar proyectos
- Asignar fechas de inicio y fin
- Asignar estado (activo, finalizado, etc.)
- Asignar departamento
- Asignar color identificativo (automático o manual)
- Registrar usuario creador del proyecto
- Invitar usuarios a proyectos
- Asignar permisos específicos a usuarios por proyecto
- Ver trazabilidad (historial de acciones)

Permisos requeridos:
- Ver proyecto
- Crear proyecto
- Editar proyecto
- Eliminar proyecto
- Invitar usuarios a este proyecto
- Asignar permisos proyecto

4.2 GESTIÓN DE ACTIVIDADES
---------------------------
Funcionalidades:
- Crear, editar, eliminar y listar actividades
- Asignar fechas de inicio y fin (con hora de fin opcional)
- Asignar estado
- Vincular actividades a un proyecto
- Soft delete (eliminación lógica)

Permisos requeridos:
- Ver actividades
- Crear actividades
- Editar actividades
- Eliminar actividades

4.3 GESTIÓN DE TAREAS
----------------------
Funcionalidades:
- Crear, editar, eliminar y listar tareas
- Asignar fechas de inicio y fin
- Asignar estado (pendiente, en progreso, completada, etc.)
- Asignar prioridad
- Asignar responsable(s)
- Tareas independientes (sin actividad asociada)
- Editar descripción de tareas
- Sistema de comentarios en tareas
- Toggle de estado (completada/pendiente)
- Toggle estado pendiente/en progreso
- Actualizar fecha de tarea
- Soft delete (eliminación lógica)

Permisos requeridos:
- Ver tarea
- Crear tarea
- Editar tarea
- Eliminar tarea

4.4 GESTIÓN DE EVIDENCIAS
--------------------------
Funcionalidades:
- Subir evidencias asociadas a tareas
- Tipos de archivos soportados:
  * Imágenes: JPG, JPEG, PNG, GIF, SVG, WEBP
  * PDFs
  * Documentos: DOC, DOCX, XLS, XLSX, PPT, PPTX
- Tamaño máximo: 10MB por archivo
- Almacenamiento en AWS S3 (bucket privado)
- Visualización de imágenes directamente en el navegador
- Visualización de PDFs en el navegador
- Descarga de documentos (Word, Excel, PowerPoint)
- Eliminación de evidencias
- URLs temporales firmadas (válidas por 60 minutos)
- Organización en carpeta "evidencias/" dentro del bucket S3

Permisos requeridos:
- Ver tarea (para ver evidencias)
- Editar tarea (para subir/eliminar evidencias)

Características técnicas:
- Integración con AWS S3 usando league/flysystem-aws-s3-v3
- Archivos privados (no accesibles públicamente)
- Generación de URLs temporales firmadas para acceso seguro
- Verificación de existencia de archivos antes de generar URLs
- Manejo de errores robusto

--------------------------------
Funcionalidades:
- Roles base del sistema (definibles)

---------------------------

- Actividad -> Tareas (HasMany)
- Tarea -> Evidencias (HasMany)
- Tarea -> Comentarios (HasMany)
- Proyecto -> Usuarios (BelongsToMany con rol)
- Usuario -> Roles (BelongsToMany)
- Rol -> Permisos (BelongsToMany)
- Usuario -> Permisos por Proyecto (HasMany a través de ProyectoUserPermiso)

14. NOTAS TÉCNICAS IMPORTANTES
================================================================================

14.1 ALMACENAMIENTO DE EVIDENCIAS
----------------------------------
- Los archivos se almacenan en AWS S3 bajo la ruta: evidencias/{nombre_archivo}
- Formato de nombre de archivo: tarea_{tareaId}_{nombre_limpio}_{timestamp}_{random}.{ext}
- Archivos privados (visibility: private)
- URLs temporales válidas por 60 minutos
- Tipos soportados: imágenes (JPG, PNG, GIF, SVG, WEBP), PDFs, documentos (DOC, DOCX, XLS, XLSX, PPT, PPTX)
- Tamaño máximo: 10MB por archivo

14.2 GESTIÓN DE PERMISOS
-------------------------
- Permisos se verifican mediante middleware CheckPermission
- Formato: CheckPermission:nombre_permiso
- Ejemplo: CheckPermission:ver proyecto
- Los permisos pueden ser globales o específicos por proyecto
- Los permisos específicos por proyecto tienen precedencia sobre los globales

14.3 MANEJO DE FECHAS
---------------------
- Actividades y tareas usan campos datetime (fecha + hora)
- Las actividades pueden tener hora de fin opcional
- Las tareas tienen fecha de inicio y fecha de fin
- El sistema maneja zonas horarias (configurable)

14.4 SOFT DELETE
----------------
- Actividades y tareas implementan soft delete
- Los registros no se eliminan físicamente, se marcan como eliminados
- Permite recuperación de datos y mantenimiento de integridad referencial
